はじめの一歩

ここでは、webの開発環境を整えて、実際にwebページを公開するところまでを説明します。
インターネットで調べれば山のように情報が出てきますが、 どれが一番いいのか、どれを信じればいいのか、よくわからないと思います。
正直なところ、僕もまだよくわかっていません。とりあえず手を動かしてみたって状態です。
ここでは、実際に僕が使っている開発環境などについて、ポイントを絞って説明していきます。
web開発初心者で、少し触ってみたいレベルの僕が説明しているので、同じレベルの人にとってはわかりやすいと思います。
それでは、早速ですが下記３点に分けて説明します。

  1. テキストエディタを用意してコードを書いてみよう
  2. 簡易的にwebページとして表示させてみよう
  3. 正式にwebページとして公開してみよう

1. テキストエディタを用意してコードを書いてみよう

web開発に必要なのはテキストエディタだけです。
テキストエディタと言われると難しく思われますが、メモ帳のことです。
ただし、普通のメモ帳を使うよりも効率的にコーディングができるように、少し機能の多いメモ帳があります。
僕が使っているのは、Atomというテキストエディタ（メモ帳）です。
詳細は調べていただければわかるのですが、Atomを使っている理由は以下の通りです。

  • とにかく無料
  • HTMLにも対応
  • 豊富なカスタム性
  • Pythonも使いやすい

ここでは、２番目と３番目について軽く触れておきます。
【HTMLにも対応】
AtomはHTMLに対応しているため、コーディングが少し楽になります。
たとえば、HTMLで大きな見出しを表示するときは<h1>見出し</h1>と記述します。
このサイトのタイトルも<h1>けんけんメモ</h1>と記述されています。
毎回<h1>見出し</h1>と打つのは、なかなかめんどくさいです。
Atomでは、h1と打てば自動的に<h1></h1>に変換してくれます。
他のタグに関しても、同様に変換してくれるため、とてもコーディングが楽になります。
また、タグの文字色が変化するため、ぱっと見でどこがタグかも理解しやすいです。
まぁこの辺は百聞は一見に如かずということで、使ってみればわかります。
【豊富なカスタム性】
Atomは、パッケージというものを追加していくことで、便利機能を簡単に追加することができます。
たとえば、”color-picker”というパッケージを追加すれば、Atom上で簡単にカラーコードを取得できます。
少しイメージしにくいですが、検索したら一発でわかりやすい解説サイトが出てくるので、そちらを参考にしてください。
僕がHTML開発で使用しているパッケージは下記の通りです。
HTML開発で必須なわけではなく、コーディング全般において少し便利なものたちだと思ってください。

  • japanese-menu：Atomを日本語化
  • file-icons：ファイルのアイコンを表示（おしゃれアイテム）
  • color-picker：カラーコードを簡単に取得
  • pigment：カラーコードの文字色をその色に変更
  • highlight-selected：選択した単語/文章をハイライト
  • minimap：ミニマップを表示（できる人感が出るから好きです）
  • minimap-highlight-selected：ミニマップ内でもハイライト

2. 簡易的にwebページとして表示させてみよう

それでは、簡単なHTML記述をして、それを簡易的にwebっぽく表示させてみましょう。
まずはじめに、"index.html"というファイルを作ってください。
適当にメモ帳なりAtomなりでファイルを新規作成して、名前をつけて保存するときに上記の名前にすれば大丈夫です。
"index"の部分は何でもいいですが、拡張子は".html"にしてください。
次に、そのファイルに下記コードをコピペしてください。
保存したファイルをダブルクリックで開いてしまうと、編集できません。
右クリックから、メモ帳なりAtomなりを指定して開いてください。 

<!DOCTYPE html>
  <html lang="ja">
    <head>
      <meta charset="UTF-8">
      <title>テスト</title>
    </head>
    <body>
      <h1>テスト表示</h1>
      <p>これが簡易的なwebページの表示です</p>
      <p>正式にwebページとして公開した際も、これと同じビジュアルになります。</p>
      <p>コーディングのデバッグとして、気軽に確認できるのは嬉しいですね！</p>
    </body>
  </html>

コピペができたら、保存して閉じましょう。
次に、先ほど保存した”index.html”をダブルクリックで開いてみましょう。
特に変な設定等をしていなければ、ブラウザで開かれると思います。
もしもブラウザで開かれない場合は、右クリックからブラウザを指定して開きましょう。
”テスト表示”という文字が無事見れましたか？
これで簡易的なwebページの表示はおしまいです。
なお、おすすめなブラウザはGoogle Chromeです。
表示したページ上で右クリックから検証を選択することで、webページの構成を確認することができます。
今後はCSSを使ってHTMLで表示した文字を装飾したりすると思います。
そこで、この検証機能が役に立ちます。
具体的には、各要素の領域やmargin, paddingの範囲を見ることができます。
これに関しても調べればわかりやすい解説ページが出てくるので、ここでお話を終わらせてもらいます。

3. 正式にwebページとして公開してみよう

さてさて、これが一番敷居が高いのではないでしょうか。
僕もここで挫折したこともありますし、最近やっとやり方を理解したところです。
その備忘録も兼ねて、この説明を書いています。
今回はGithub Pagesを利用します。
Githubはコマンドプロンプトを使って、ローカルから簡単に作業できます。
そのほうが効率的だしかっこいいのですが、何も理解していないのにそれをやるのには拒否反応が出てしまいました。
そのため、コマンドプロンプトは使わない方法を説明していきます。
また、ここでは複数のページを公開するところまで説明していきます。
複数のページの公開をコマンドプロンプトなしで説明しているのは、おそらくここだけでしょう。（弊社調べ）
下記の流れで説明していきます。

  1. Githubに登録
  2. リポジトリを新規作成
  3. ソースコードをアップ
  4. URLを確認
  5. 複数ページを用意する方法

順を追って説明しますね。
【Githubに登録】
はい、迷わず検索しましょう。
何も難しいことはないです。
ひとつ言えば、僕はつい最近ここでつまづきました。
IDやアドレスを入れて登録するのですが、何回やっても同じ承認ページを繰り返す現象がおきまして、、、。
結局理由は分かりませんが、Safariでやっていたことが原因でした。
Chromeでは一発で登録できたので、Safariの方はご注意ください。
【リポジトリを新規作成】
こちらもレッツ検索です。
悩むのが、Repository nameですね。
IDと同じにしなさいって書いてあったり、なんでもいいよって書いてあったり、正解がわかりません。
今回は、とりあえず”ID.io”にしてみましょう。
僕は上手く動けば問題ないので、特に他のRepository nameで試したりはしていません。
Repository name以外の部分はノータッチでかまいません。
Create repositoryを押してから承認作業をすれば、登録完了です。
メールアドレスの確認くらいは適当に済ましておいてください。
【ソースコードをアップ】
こちらも調べたらすぐに出てきますが、簡単に説明します。
"<> Code"をクリックすると、これまでアップロードしたファイルを一覧で見ることができます。
当然、まだ何も表示されません。
右上の"Add file"から"Upload files"を選択します。
あとは表示されている通りにファイルをアップロードして、"Commit changes"をクリックすれば終了です。
まずは、"index.html"をアップロードしてみてください。
【URLを確認】
こちらも調べたら、、、なので、簡単に説明します。
"Setting"を開き、下の方にスクロールしていくと"Github Pages"というのがあります。
そのすぐ下にある"Your site is published at URL"とあります。
このURLが、先ほどアップロードした"index.html"を公開しているwebサイトのURLになります。
ここまでくると、一通りはできたかな、と思います。
お疲れ様でした。
【複数ページを用意する方法】
これまで説明した通りに忠実にやっていただけている場合に限ります。
違う状況だと、僕が試していないので上手くいくのかわかりません。
"index.html"という名前は、いわばwebのTOPページに当たります。
URLは"https://ID.github.io"となっているはずです。
たとえば、新たに"test.html"というファイルをアップロードします。
そうすると、"https://ID.github.io/test"というURLが、"test.html"に対応しています。
同じ法則で、追加したhtmlファイルに対応したURLが生成されていきます。
どこでこのURLを確認するのかはわかりません。
コマンドでいろいろやってるGithub Pasesの紹介ページを部分的にかいつまんで、もしかしたら！と試したら想定通りだったというだけです。
とりあえず、複数ページを用意するのがとても簡単だと言うことがわかりましたね。
あとはCSSのアップロードや画像のアップロードもありますが、同じ手順でアップロードすればOKです。
細かいところは調べてみてください、すぐにヒットします。
最後まで読んでいただきありがとうございました。
